# 技术架构说明

## 1. 架构定位

本项目采用前后端分离思路进行设计，前端负责页面展示、用户交互和数据可视化，后端负责身份认证、权限校验、工单业务逻辑、数据库操作和 AI 接口调用。

项目虽然是本地运行版本，但整体结构模拟企业内部管理系统的常见设计方式：

- 用户通过浏览器访问系统
- 前端页面调用后端 REST API
- 后端通过 JWT 识别用户身份
- 后端根据角色进行权限控制
- 后端将业务数据存入 SQLite 数据库
- AI 模块对工单内容进行摘要、优先级判断和处理建议生成

## 2. 分层结构

```text
用户浏览器
  │
  ▼
前端展示层 public/
  ├── 登录注册页面
  ├── 数据看板页面
  ├── 工单列表页面
  ├── 工单详情弹窗
  └── 创建工单页面
  │
  ▼
后端服务层 server.js + routes/
  ├── Auth API
  ├── User API
  ├── Ticket API
  ├── Dashboard API
  └── AI API
  │
  ▼
业务与权限层 middleware/ + routes/
  ├── JWT 身份认证
  ├── 角色权限校验
  ├── 工单可见范围控制
  └── 工单状态流转控制
  │
  ▼
数据访问层 db.js
  ├── users
  ├── tickets
  ├── comments
  └── logs
  │
  ▼
第三方服务层 utils/ai.js
  ├── DeepSeek API
  └── 本地 AI 模拟规则
```

## 3. 角色权限设计

| 角色 | 主要权限 |
|---|---|
| 员工 Employee | 创建工单、查看自己的工单、添加补充说明、关闭自己的工单 |
| 技术人员 Technician | 查看未分配或分配给自己的工单、更新状态、添加处理记录、使用 AI 辅助 |
| 管理员 Admin | 查看全部工单、分配技术人员、更新状态和优先级、查看用户列表和完整统计看板 |

## 4. 工单状态流转

```text
pending 待处理
   │ 管理员分配 / 技术人员接入
   ▼
processing 处理中
   │ 问题处理完成
   ▼
resolved 已解决
   │ 员工确认 / 管理员关闭
   ▼
closed 已关闭
```

## 5. AI 模块设计

AI 模块不是独立聊天机器人，而是服务于工单业务流程：

- 工单摘要：帮助技术人员快速理解问题
- 优先级推荐：辅助判断问题紧急程度
- 处理建议：给出可执行排查方向

如果配置了 DeepSeek API Key，系统会调用真实 API；如果没有配置，系统会使用本地规则生成演示结果，保证项目在无 Key 情况下也可以完整运行。
