# 代码学习指南：从看懂项目到能讲清楚

## 1. 不要一开始看每一行代码

这个项目比普通学生作业复杂，所以不要一开始就逐行看代码。正确顺序是：

```text
先看项目结构
再看功能流程
再看核心文件
最后看具体代码细节
```

你的目标不是背代码，而是能讲清楚项目如何运行。

## 2. 项目文件结构

```text
enterprise-ticket-management-system/
├── public/              前端页面文件
│   ├── index.html        页面结构
│   ├── styles.css        页面样式
│   └── app.js            前端交互和请求接口

```

## 4. 核心流程一：登录

登录流程可以这样理解：

```text
用户输入邮箱和密码
前端发送登录请求
后端查询数据库中的用户
密码正确后生成 JWT Token
前端保存 Token
之后访问接口时带上 Token
后端通过 Token 判断用户身份
```

答辩时可以说：

> 系统使用 JWT 实现登录认证。用户登录成功后，后端会生成 Token，前端保存 Token，并在后续请求中携带 Token，后端通过中间件验证用户身份。

## 5. 核心流程二：创建工单

创建工单流程：

```text
员工填写标题、描述、分类和优先级
前端调用创建工单接口
后端验证用户身份
后端把工单保存到 tickets 表
返回创建成功结果
前端刷新工单列表
```

答辩时可以说：

> 用户提交工单后，前端会通过 POST 请求将数据发送到后端，后端完成身份验证后将工单写入数据库，并返回结果给前端展示。

## 6. 核心流程三：角色权限

三种角色：

```text
employee：员工，只能提交和查看自己的工单
technician：技术人员，可以处理分配给自己的工单
admin：管理员，可以查看全部工单、分配工单和管理用户
```

答辩时可以说：

> 系统通过角色字段区分用户权限，并在后端接口中进行权限校验，避免普通员工访问管理员功能。

## 7. 核心流程四：AI 辅助

AI 功能流程：

```text
用户点击 AI 辅助按钮
前端把工单内容发送给后端
后端调用 AI 工具函数
如果有 API Key，就请求 DeepSeek API
如果没有 API Key，就返回本地模拟建议
前端展示摘要、优先级和处理建议
```

答辩时可以说：

> AI 功能不是独立聊天，而是嵌入工单处理流程中，用于生成工单摘要、推荐优先级和提供处理建议，使项目更贴近企业服务台场景。

## 8. 你至少要会讲的 5 个文件

### server.js

项目启动入口，负责启动 Express 后端服务，并把前端页面和后端接口连接起来。

### src/db.js

数据库文件，负责创建用户表、工单表、评论表和日志表，并初始化演示数据。

### src/routes/auth.js

登录注册接口文件，负责用户注册、用户登录和 Token 生成。

### src/routes/tickets.js

工单核心接口文件，负责工单创建、查询、分配、状态更新和评论记录。

### public/app.js

前端核心交互文件，负责按钮点击、表单提交、调用后端接口和更新页面显示。

## 9. 学习目标

学完这个项目后，你要能做到：

- 说清楚项目解决什么问题
- 说清楚三种角色的权限
- 说清楚工单从创建到完成的流程
- 说清楚前端、后端、数据库如何配合
- 说清楚 AI 功能接入在哪里
- 说清楚这个项目和企业 Service Desk / ITSM 的关系
