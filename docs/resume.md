# 简历与申请材料写法

## 1. 简历项目经历版本

**Enterprise Ticket Management System 企业内部工单管理系统**  
**项目类型：企业业务场景模拟项目 / 全栈 Web 开发项目**  
**技术栈：HTML、CSS、JavaScript、Node.js、Express、SQLite、ECharts、JWT、DeepSeek API**

基于企业内部 IT 服务台场景，设计并开发了一个工单管理系统，实现员工提交工单、技术人员处理工单、管理员分配工单和查看数据统计等功能。系统采用前后端分离思路，后端基于 Node.js 和 Express 提供 RESTful API，数据库使用 SQLite 存储用户、工单、处理记录和操作日志等数据。

项目设计了员工、技术人员和管理员三类角色，并根据不同角色实现差异化权限控制。员工可以创建和查看自己的工单，技术人员可以处理分配给自己的工单，管理员可以查看全部工单、分配处理人员并查看统计看板。系统还集成了 AI 辅助功能，可根据工单描述自动生成问题摘要、推荐优先级并提供处理建议。

通过该项目，我熟悉了企业后台系统的基本业务流程，掌握了用户认证、角色权限、数据库设计、接口开发、数据统计和 AI API 接入等实践内容，提升了自己面向真实业务场景进行系统设计和开发的能力。

通过这个项目，我不仅加深了对 Web 系统开发流程的理解，也学习了如何从业务需求出发进行功能设计、数据库设计和接口开发。相比单纯的页面练习，该项目更重视业务流程、权限边界和数据管理，让我对企业信息系统的开发方式有了更具体的认识。同时，我也尝试将 AI 能力应用于工单摘要、优先级判断和处理建议生成，进一步增强了我对 AI 技术落地应用的兴趣。

## 4. 面试介绍稿

我做过一个企业内部工单管理系统，项目背景是模拟公司 IT 服务台的日常工作流程。员工在系统中提交账号、网络、硬件或业务系统问题，技术人员负责处理工单，管理员可以分配工单并查看数据统计。

这个项目主要包括登录注册、角色权限、工单创建、工单分配、状态更新、处理记录、数据看板和 AI 辅助建议等功能。后端使用 Node.js 和 Express，数据库使用 SQLite，前端使用 HTML、CSS、JavaScript 和 ECharts。

我认为这个项目最大的价值是让我从业务场景出发理解系统开发，而不是只做页面。通过项目，我学习了企业后台系统常见的权限控制、业务流转、数据库设计和接口开发方式，也尝试把 AI 能力嵌入实际业务流程中。
