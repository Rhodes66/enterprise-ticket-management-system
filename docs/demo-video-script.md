# 项目演示视频脚本

建议录制一个 2-3 分钟的视频，用于作品集或面试前展示。

## 1. 开场介绍（20 秒）

大家好，这是我完成的 Enterprise Ticket Management System 企业内部工单管理系统。这个项目模拟企业 IT 服务台场景，用于管理员工提交的问题、技术人员处理流程以及管理员的数据统计。

## 2. 登录页面（20 秒）

系统提供登录功能，并内置了管理员、技术人员和员工三种演示账号。不同角色登录后可以看到不同功能，这体现了系统的角色权限设计。

## 3. 管理员功能演示（40 秒）

现在我使用管理员账号登录。管理员可以查看数据看板，包括总工单数、待处理工单、已完成工单等统计信息。管理员还可以查看所有工单，并将工单分配给技术人员。

## 4. 员工提交工单（30 秒）

员工登录后，可以创建新的工单，填写问题标题、描述、分类和优先级。提交后，工单会保存到数据库，并出现在工单列表中。

## 5. 技术人员处理工单（30 秒）

技术人员可以查看分配给自己的工单，更新工单状态，并添加处理记录。这样可以模拟企业中问题从提交到处理再到关闭的完整流程。

## 6. AI 辅助功能（30 秒）

系统还加入了 AI 辅助功能，可以根据工单描述生成问题摘要、推荐优先级并提供处理建议。这个功能的作用是帮助技术人员更快理解问题并提高处理效率。

## 7. 总结（20 秒）

通过这个项目，我学习了企业后台系统开发中的用户认证、角色权限、数据库设计、接口开发、数据统计和 AI API 集成。这个项目也帮助我弥补了实习开发经历不足的问题，提升了我的软件工程实践能力。
