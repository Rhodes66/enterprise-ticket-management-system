# 项目截图指南

申请作品集和 GitHub README 里建议放 6-8 张截图。截图不需要太多，但要体现系统完整性。

## 1. 登录页面

**截图内容：** 登录框、系统名称、企业工单系统标题。  
**说明文字：** 用户通过邮箱和密码登录系统，系统会根据用户角色展示不同功能。

## 2. 管理员数据看板

**截图内容：** 总工单数、待处理数、状态分布图、优先级分布图、趋势图。  
**说明文字：** 管理员可以通过数据看板查看企业内部工单处理情况和技术人员工作量。

## 3. 工单列表页面

**截图内容：** 工单列表、状态、优先级、负责人、筛选条件。  
**说明文字：** 系统支持按状态、优先级和关键词筛选工单，方便不同角色快速定位问题。

## 4. 创建工单页面

**截图内容：** 标题、问题类型、优先级、问题描述输入框。  
**说明文字：** 员工可以提交 IT 问题，系统会保存工单信息并进入后续处理流程。

## 5. 工单详情页面

**截图内容：** 工单详细描述、处理记录、状态更新、评论区。  
**说明文字：** 技术人员可以在详情页查看问题背景、添加处理记录并更新工单状态。

## 6. AI 辅助建议页面

**截图内容：** AI 摘要、优先级建议、处理建议。  
**说明文字：** 系统集成 AI 辅助功能，用于生成工单摘要、推荐优先级和提供处理思路。

## 7. 用户管理页面

**截图内容：** 用户列表、角色信息。  
**说明文字：** 管理员可以查看系统用户，体现后台系统中的用户与角色管理能力。

## 8. 项目运行终端

**截图内容：** npm start 成功运行、localhost 地址。  
**说明文字：** 项目可以在本地完整运行，说明代码和依赖配置是可复现的。
