# V2 升级计划：让项目更像企业实习项目

## 1. 为什么要做 V2

当前版本已经可以运行并展示核心功能，但如果希望更接近企业真实项目，可以继续加入一些企业开发中常见的功能和工程配置。

## 2. 推荐升级功能

### 2.1 附件上传

员工提交工单时可以上传截图或文件，例如错误截图、系统日志或说明文档。

企业价值：真实 IT 工单系统通常支持附件。

### 2.2 SLA 超时提醒

根据工单优先级设置处理时限，例如：

```text
紧急：4 小时内处理
高：8 小时内处理
中：24 小时内处理
低：72 小时内处理
```

企业价值：体现服务质量管理思维。

### 2.3 周报导出

管理员可以一键生成本周工单处理报告，包括工单数量、完成率、平均处理时长和高优先级问题。

企业价值：更接近公司管理后台。

### 2.4 邮件通知

工单被分配或状态更新时，系统可以发送邮件通知相关用户。

企业价值：模拟真实业务通知流程。

### 2.5 Docker 部署

加入 Dockerfile 和 docker-compose.yml，让项目可以用容器方式启动。

企业价值：体现部署和工程化能力。

### 2.6 API 测试

加入 Postman Collection 或自动化测试脚本。

企业价值：体现测试和接口验证能力。

## 3. V2 简历增强写法

如果完成 V2，可以在简历中加一句：

> 后续进一步扩展附件上传、SLA 超时管理、周报导出和 Docker 部署配置，使项目更接近企业内部服务台系统的实际开发流程。

## 4. 推荐开发顺序

```text
第一步：附件上传
第二步：SLA 超时字段和状态提醒
第三步：周报导出
第四步：Docker 部署
第五步：API 测试文档
```

不要一次性做太多，按模块逐步完成。
