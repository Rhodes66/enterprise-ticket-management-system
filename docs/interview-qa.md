# 面试 / 答辩常见问题准备

## Q1：你为什么做这个项目？

我做这个项目主要是为了补强自己在企业级 Web 系统开发方面的实践能力。普通课程项目往往只关注页面和简单增删改查，而企业工单系统更接近真实业务场景，涉及用户角色、权限控制、业务流程、数据库设计、数据统计和 AI 辅助功能，因此更能锻炼完整系统开发能力。

## Q2：这个项目解决什么问题？

它模拟企业内部 IT 服务台的问题处理流程。员工可以提交电脑、网络、账号或业务系统问题；技术人员负责处理工单；管理员可以分配工单并查看处理效率。系统的目标是让企业内部问题处理流程更加清晰、可追踪、可统计。

## Q3：你的项目技术架构是什么？

项目采用前后端分离思路。前端使用 HTML、CSS、JavaScript 和 ECharts，负责页面展示、用户交互和数据可视化。后端使用 Node.js 和 Express，负责登录注册、权限校验、工单管理、数据统计和 AI 辅助接口。数据库使用 SQLite 存储用户、工单、评论和日志数据。

## Q4：你在项目里负责什么？

这个项目是我的个人全栈项目，我参与了从需求设计、功能规划、数据库设计、接口开发、页面实现到项目运行测试的完整过程。重点包括角色权限设计、工单流转逻辑、数据表设计、接口开发和 AI 辅助功能接入。

## Q5：为什么设计三种角色？

因为企业后台系统通常不同用户有不同权限。员工只需要提交和查看自己的工单；技术人员负责处理工单；管理员需要管理全部工单、分配人员并查看数据统计。这样设计可以更贴近真实企业系统，也能体现权限控制思维。

## Q6：数据库设计有哪些表？

主要有四张表：users 表存储用户信息和角色；tickets 表存储工单标题、描述、状态、优先级和负责人；comments 表存储处理记录；logs 表记录关键操作。这样的设计可以支持完整的工单流转和后续追踪。

## Q7：AI 功能在项目中有什么作用？

AI 不是作为普通聊天机器人存在，而是服务于工单业务。它可以根据工单描述生成问题摘要、推荐工单优先级，并给技术人员提供处理建议。这样 AI 功能和企业业务流程结合得更紧密。

## Q8：这个项目最大的难点是什么？

最大的难点是把不同模块组织成一个完整业务流程，而不是只做单个功能。比如登录后要识别用户角色，不同角色要看到不同页面，工单状态要合理流转，管理员还要能通过看板查看统计结果。这些都需要从整体系统角度进行设计。

## Q9：项目还有哪些可以改进的地方？

后续可以继续加入附件上传、邮件通知、SLA 超时提醒、工单导出、Docker 部署、MySQL 数据库、操作日志管理和更完整的权限模型。如果继续优化，它可以发展成更接近企业真实使用的 ITSM 系统。

## Q10：这个项目对你申请研究生有什么帮助？

这个项目帮助我把软件工程知识应用到企业业务场景中，也让我理解了 AI 技术如何落地到具体流程中。它体现了我在 Web 开发、数据库设计、权限控制、数据可视化和 AI API 应用方面的实践能力，也说明我在本科基础之外主动补强了自己的项目经验。
