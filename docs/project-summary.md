# 项目总结

## 1. 项目背景

在企业日常运营中，员工经常会遇到账号权限、网络连接、硬件设备、业务系统等问题。如果这些问题仅通过聊天工具或口头沟通处理，容易出现记录混乱、责任不清、处理进度不可追踪等情况。因此，企业内部通常需要一个工单系统来规范问题提交、分配、处理和统计流程。

本项目基于这一企业真实场景，设计并开发了一个企业内部工单管理系统，用于模拟员工提交问题、技术人员处理问题、管理员管理工单和查看数据统计的完整流程。

## 2. 完成内容

本项目实现了以下核心功能：

- 用户注册与登录
- JWT 身份认证
- 员工、技术人员、管理员三类角色权限
- 工单创建、查询、筛选和详情查看
- 工单分配、状态更新和优先级管理
- 工单处理记录添加
- 数据统计看板
- ECharts 图表展示
- AI 工单摘要、优先级推荐和处理建议
- SQLite 数据持久化

## 3. 技术收获

通过本项目，我进一步理解了企业后台系统的基本开发流程，包括需求分析、模块划分、数据库设计、接口开发、权限控制和前后端联调。

在后端方面，我使用 Node.js 和 Express 设计 RESTful API，并通过 JWT 实现用户身份认证；在数据库方面，我设计了用户表、工单表、处理记录表和操作日志表，完成了基础数据持久化；在前端方面，我实现了登录注册、工单列表、详情弹窗、创建表单和数据看板页面；在 AI 应用方面，我将 AI 能力与具体业务场景结合，用于辅助工单摘要、优先级判断和处理建议生成。

## 4. 项目亮点

本项目与普通课程练习不同，它不是单纯展示页面，而是围绕企业内部 IT 服务台业务流程进行设计，包含明确的业务角色、权限边界、数据流转和统计分析功能。项目中的 AI 能力也不是简单聊天，而是嵌入到工单处理流程中，提升技术人员理解问题和处理问题的效率。

## 5. 不足与改进方向

当前项目仍然是一个本地运行的 MVP 版本，后续可以继续完善：

- 增加文件上传，支持用户上传错误截图
- 增加邮件通知或站内消息提醒
- 增加 SLA 超时提醒机制
- 增加 Docker 部署配置
- 使用 MySQL 或 PostgreSQL 替换 SQLite
- 使用 Vue 或 React 重构前端
- 增加接口测试和单元测试
- 增加日志审计和安全校验

## 6. 申请材料中的表达

这个项目可以用于证明我具备一定的软件工程实践能力，能够围绕企业业务需求进行系统设计和开发。它也体现了我对 AI 应用落地的理解：AI 不只是独立聊天工具，也可以嵌入企业业务流程，用于提高信息处理、问题分类和决策辅助的效率。
